The book of Magos

Superiorix

Acting as our tutor in making games is Superiorix the head wizard. He will occasionally give instructions or tips on how to create games.

Principes

Principes is the designer wizard of Magos. His job is to design the game by dragging components others have made into correct positions in the game grid. He also takes care of texts in the game, score handling, game time and such.

Font

Define the font family, size, color and background color for texts. Usable for example with components containing text or dialog and with scene components like clock, scoreboard and highscore.

Score

Attach a score change event into a component with a rule, like "Get 100 points when this collision event occurs"

Dialog

Attach a dialog into a component with a rule. With dialog you can create chained texts (see Text tool) i.e. once the first text is red the second appears until all the texts have been seen through. You can also define who says the text and have player character get into conversations with non-player characters.

Text

Attach a text to a component with a rule. Text is displayed as a popup once the rule triggers. The text is visible until the player closes it.

Physicus

Physicus is a physician who makes components act the way they do. This includes setting component types, creating rules and adjusting gravity.

Controls

Choose how the player character moves. There are predefined control types of which you can pick the one that suits your game. Currently there are two types available "twoway" and "fourway".

Collision

Attach a collision rule to a component with another component. Collision event triggers an action when the component containing the collision rule hits with another component set as target in the rule. "Player collides with a Trap which causes player to lose 10 health".

Gravitation

Component can be set to have gravity affecting it. This means that the component will fall downwards unless there is some impassable component blocking the fall. The bigger the gravity value, the faster the fall. Gravity can be set as inverted which makes the fall happen upwards.

Type

Select a general type for a component. This sets some features of that component. For example type "block" makes the component solid and unpassaple, "collectible" means that the component disappears once player character collides with it (possibly giving points to the player in the process).

Artifex

Game graphics are created by Artifex the artist. Graphics can be either normal pictures or animations.

Image

Attach an image to a component. If this is not done, the component will be invisible to the player. The image is either done by oneself with the drawing tool or selected from the shared image pool. Image can be attached also as a background image to the game, intro screen or outro screen.

Animation

Component textures can have frames for animation included. It is possible to play an animation when a rule is triggered. For example "Player moves left, play animation (which shows the character walking)" A component can have only one animation available at any time as there can only be one image included in a component and the animation frames are included in the image. However it is possible to change the component's image with a rule so an image with suitable animation can be activated if needed. Animation can be run once or looped for several or infinite times.

Musicus

Magos has a composer called Musicus. He creates all the music and sound effects used in the game.

Audio

Attach a sound to a component with a rule. Alternatively a sound can be attached to a background scene component as a background music. Sounds are recorder using the recording tool or selected from the shared sounds pool.
